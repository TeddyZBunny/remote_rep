cd путь

	В этой папке надо все отслеживать
git init - делает папку master и создается папка .git


	Добавление файлов
git add .  добавит все файлы
git add Git.txt добавит файл
добавлять нужно перед каждым коммитом

	Подтверждение изменений
git commit -m "Added file"
git commit если без комментария в винде откроется вин, чтобы выйти надо нажать Esc, потом ввести :q 
git commit -am "What u did" в случае, если комиитать нечего, но ты просто изменил содержание файла, то пишем также

	Просмотр истории имзенений
git log
пробелы листают логи вниз, чтобы выйти, надо дойти до конца, там будет END, нажимаем q

	Как вернуться к опредленнмоу комиту или состоянию файла
git checkout 67bfda4219e1d4cd222e549f5953938c6206d353 - это id из лога

git checkout отменяет все локальные незаконченные изменения
git switch -  переключает на ветку
git checkout Git.txt  отменить определенное локальное незакоммиченное изменение


	Возвращение к старому файлу или возварщение удаленного файла
git restore Py.txt
git restore --staged  Py.txt - опция staged должна работать с файлами, которые добавлены в индекс и подготовлены к коммиту

	Отменить коммит
git reset --mixed переключите HEAD на последний коммит, и все изменения, добавленные после него, будут доступны в качестве неотслеживаемых изменений в вашем каталоге
git reset --soft переключается на последний коммит, однако, изменения, добавленные после этой фиксации, остаются с пометкой staged
git reset --hard  переключите HEAD на последний коммит и уничтожите изменения, сделанные после него.
git reset head Git.txt отмена незакомиченных изменений
git revert head отмена предыдущего коммита, создавая новый комит

	Создание ветки
git checkout -b my создаем ветку оттуда откуда находимся

	Переход в ветку
git checkout [название ветки] перейти в ветку

	Сливание в мастер ветку и наоборот
переходим в ветку и пишем git merge master
переходим в мастер и пишем git merge название_ветки
	
	Клонирование из удаленного репозитория на локальное устройство

git clone ссылка ssh из гитхаба

	Проверка и получение новох изменений в УР
git pull

git push --set--upstream название_репозитория название ветки
